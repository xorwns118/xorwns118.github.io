# 테스트 코드란?

- `Test Code`란 **소프트웨어의 기능과 동작을 테스트하는 데 사용 되는 코드**
- 소프트웨어 테스트는 소프트웨어의 결함을 찾아내고 수정하는 과정에서 매우 중요
- `Test Code` 는 개발자가 작성한 코드를 실행하고 예상된 결과가 나오는지 확인하는 데 사용

### 작성하는 이유?

- 문서화 역할
- 코드에 결함을 발견하기 위함
- 리팩토링 시 안정성 확보
- 테스트 하기 쉬운 코드를 작성하다 보면 더 낮은 결합도를 가진 설계를 얻을 수 있음

### TDD

- `Test Driven Development` (테스트 주도 개발)
- 프로덕션 코드보다 테스트 코드를 먼저 작성하는 개발 방법
- TFD(Test First Development + 리팩토링
- 기능 동작을 검증 (메소드 단위)

### BDD

- `Behavior Driven Development` (행위 주도 개발)
- 시나리오 기반으로 테스트 코드를 작성하는 개발 방법
- 하나의 시나리오는 `Given`, `When`, `Then` 구조를 가짐

### 테스트를 잘 하기 위한 기반

- 클래스나 메소드가 `SRP` 를 잘 지키고, 크기가 적절히 작아야 함
    - 테스트를 집중력있게 만들 수 있고 한 메소드에 너무 많은 테스트를 수행하지 않아도 됨
    - 테스트를 하는 것의 장점이기도 함 (테스트를 하면 자연스럽게 역할이 확인되면서 쪼개짐)
- 적절한 `Mocking` 을 통한 격리성 확보
    - 단위테스트가 만능은 아니지만, 위의 `SRP` 처럼 해당 메소드의 역할을 정확이 테스트 하려면 주변 조건을 적절히 통제해야 한다.
- 꼼꼼히 테스트 && 과도하게 많은 테스트와 코드가 생기지 않도록 적절히 끊기
    - `Test Code` 도 코드 리뷰 시 적절한 테스트를 하는지 확인 필요
- 테스트 코드 개선을 위한 노력
    - `Test Code` 도 리팩토링 필요
    - `Test Code` 의  기법도 지속적인 고민 필요(통합 테스트 등)

- `Junit`
    - `Junit` : `xUnit` 아키텍쳐를 가져와 `Java`의 유닛 테스트를 위한 프레임워크
    - `Junit` 자체는 온전히 `Spring` 을 위한 유닛 테스트가 아님 ⇒ `Java` 를 위한 테스트
    - Class 에서 `Cmd` + `Shift` + `T` ⇒ Create New Test
- `Mockito`
    - 가짜 동작을 정해 놓는것
